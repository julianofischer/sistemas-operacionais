No princípio

  • Cada computador tinha seu próprio S.O.
  • Customizado para servir a um propósito específico
  • Escrito em assembly
    • Não portável

1969 - UNIX

  • Novo sistema operacional
  • Ken Thompson e Dennis Ritchie (AT&T Bell Labs)
  • Escrito em C
  • Portável (várias plataformas de hardware)

Anos 80

  • Várias companhias desenvolviam seus próprios sistemas UNIX
  • Desenvolvimento de vários sistemas UNIX comerciais
  • Surgimento dos PCs
  • Alguns sistemas UNIX capazes de rodar em PCs
  • Desempenho não satisfatório

Anos 80 - GNU

  • GNU is not UNIX!
  • Objetivo: sistema operacional livremente acessível para qualquer um
  • Várias ferramentas de linha de comando do Linux hoje são GNU

Anos 80 - FSF

  • Free Software Foundation
  • Objetivo: eliminar restrições sob cópia, estudo e modificação de softwares
  • Atualmente dedica-se mais aos aspectos legais

1991 - Linus e o Linux

  • Objetivo: desenvolver um sistema compatível com UNIX e capaz de rodar nos PCs da época

Afinal, o que é o Linux?

  • Um sistema operacional
  • Gerencia todos os recursos relacionados ao hardware
  • Faz a comunicação entre software e hardware
  • Bootloader + kernel + daemons + shell + graphical server + applications = Linux

Bootloader

  • Software que gerencia o processo de inicialização do computador

Bootloader

Bios vs UEFI

Kernel (núcleo)

  • Parte que é efetivamente chamada "Linux"
  • Parte de mais baixo nível do S.O.
  • Gerencia CPU, memória e periféricos

Daemons

  • Serviços de background que iniciam durante o boot ou após o login
  • sshd: espera por conexões SSH
  • syslogd: gerencia sessões
  • cron: executa tarefas agendadas

Shell

  • Permite controlar o S.O. via comandos em uma interface de texto
  • "Pega" comandos do teclado e passa para o S.O.

Servidor Gráfico

  • Sistema que exibe os gráficos no monitor
  • X, servidor X, sistema de janelas X, X-window, X11

Ambientes Desktop

  • Provê uma interface gráfica completa
  • Geralmente, cada uma contém aplicações "embutidas"

General Public License

Baseia-se em 4 liberdades:

0. Executar o programa para qualquer propósito

1. Estudar e adaptar o programa as suas necessidades

2. Liberdade de redistribuir cópias do programa

3. Aperfeiçoar o programa e liberar os seus aperfeiçoamentos

Distribuições Linux

  • Coleção de software baseada no Kernel Linux
  • Geralmente contém um sistema de gerenciamento de pacotes, um ambiente desktop, bibliotecas e ferramentas GNU, etc...
  • Disponível para uma ampla variedade de sistemas
    • Roteadores: OpenWrt, DD-WRT
    • Supercomputadores: Rocks Cluster Distribution
    • PCs: Mint, Ubuntu, Fedora, Arch Linux...
